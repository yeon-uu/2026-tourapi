# 시즌별 카드 컬렉션 기능 설계

> AI-DO의 재방문 유도 + 확장성 핵심 기능

---

## 1. 컨셉

같은 역이라도 **시즌마다 다른 카드**를 수집할 수 있다.
봄의 벚꽃 카드와 겨울의 설경 카드는 다르다.
시즌 카드를 모으기 위해 다시 찾아가는 것, 그게 지역 관광 활성화의 자연스러운 동기이다.

---

## 2. 시즌 구분

### MVP (공모전용): 날짜 기반 자동 판정

| 시즌 | 기간 | 아이콘 |
|------|------|--------|
| 봄 (spring) | 3월 ~ 5월 | 🌸 |
| 여름 (summer) | 6월 ~ 8월 | 🌊 |
| 가을 (autumn) | 9월 ~ 11월 | 🍂 |
| 겨울 (winter) | 12월 ~ 2월 | ❄️ |

### 확장 비전: TourAPI 축제 데이터 기반

벚꽃축제 기간 = 봄 시즌, 해수욕장 개장 = 여름 시즌 등
`searchFestival1` API와 연동하여 동적 시즌 판정 가능.

## 3. 카드 이미지 우선순위

```
1순위: 콜라보 일러스트 (시즌 한정)
2순위: TourAPI firstimage 사진 (기본값)
```

| 카드 유형 | 이미지 소스 | 설명 |
|----------|-----------|------|
| **일반 카드** | TourAPI `firstimage` URL | 모든 역에 기본 제공 |
| **콜라보 카드** | 일러스트레이터 작업물 URL | 시즌 한정, 지자체/작가 협업 |
| **축제 한정 카드** | 축제 관련 이미지 URL | 특정 축제 기간에만 획득 가능 |

### 콜라보 카드 예시

```
┌─────────────────────┐
│ ❄️ 겨울 한정          │  ← 시즌 배지
│ ┌─────────────────┐ │
│ │                 │ │
│ │  일러스트레이터   │ │  ← 콜라보 이미지
│ │  가 그린 삼척역   │ │
│ │  설경 일러스트    │ │
│ └─────────────────┘ │
│ 삼척역              │
│ 동해산타열차  SSR    │
│ illust. @작가명      │  ← 작가 크레딧
│ 2026.01.15         │
└─────────────────────┘
```

## 4. DB 변경

### stamps 테이블

```sql
-- 기존 컬럼에 추가
season        VARCHAR(10) NOT NULL,  -- 'spring' | 'summer' | 'autumn' | 'winter'
card_image_url VARCHAR(500),          -- TourAPI firstimage 또는 콜라보 이미지 URL

-- UNIQUE 제약 변경
-- 기존: UNIQUE(user_id, station_id)       → 역당 1개
-- 변경: UNIQUE(user_id, station_id, season) → 역+시즌당 1개
```

### 시즌 판정 로직 (백엔드)

```python
from datetime import date

def get_current_season() -> str:
    month = date.today().month
    if month in (3, 4, 5):
        return "spring"
    elif month in (6, 7, 8):
        return "summer"
    elif month in (9, 10, 11):
        return "autumn"
    else:
        return "winter"
```

## 5. 페이지별 역할 재정리

| 페이지 | 카드 이미지 | 사용자 사진 업로드 |
|--------|-----------|-----------------|
| **가챠 뽑기** | TourAPI 사진 or 콜라보 일러스트 | ❌ |
| **미션 수행** | - | ❌ |
| **스탬프 획득** | 수집 카드 표시 (TourAPI/콜라보) | ❌ |
| **공유 페이지** | 카드 + 내 사진 합성 | ✅ (여기서만) |
| **아카이브** | 역별 시즌 카드 갤러리 | ❌ |

### 핵심 설계 의도

- **수집 카드** = 우리가 제공하는 이미지만 사용 (TourAPI / 콜라보)
- **공유용 카드** = 사용자 사진 업로드 허용 (SNS 인증용)
- 서버에 사용자 사진 저장 불필요 (localStorage만 사용) → 서버 비용 최소화

## 6. 아카이브 시즌 카드 뷰

### 기존

```
삼척역 [🔓] → 스탬프 카드 1장
```

### 시즌카드 적용 후

```
삼척역 [🔓]
├── 🌸 봄 카드    (미보유)
├── 🌊 여름 카드   (미보유)
├── 🍂 가을 카드   (미보유)
└── ❄️ 겨울 카드   (보유 ✅)
```

- 보유 카드: 탭하면 카드 이미지 확대 표시
- 미보유 카드: 흑백 실루엣 + 자물쇠 아이콘

## 7. 가챠 중복 정책 변경

### 기존

같은 역은 다시 뽑히지 않음 (스탬프 발급된 역 제외)

### 시즌카드 적용 후

**같은 역이라도 현재 시즌에 카드가 없으면 다시 뽑힐 수 있음**

```python
# 제외 조건 변경
# 기존: 스탬프가 있는 station_id 전부 제외
# 변경: 현재 시즌 + 해당 station_id 스탬프가 있는 경우만 제외
excluded = stamps.filter(season == current_season).station_ids
```

## 8. 비용 구조

| 항목 | 비용 | 이유 |
|------|------|------|
| 카드 이미지 저장 | **0원** | TourAPI 이미지 URL 참조 (우리 서버 저장 X) |
| 콜라보 이미지 | **0원 (MVP)** | 확장 시에만 필요, 현재는 TourAPI로 대체 |
| 사용자 사진 | **0원** | localStorage만 사용, 서버 업로드 없음 |
| LLM 호출 | **미미** | 시즌 변경으로 추가 호출 없음 (미션 생성은 1회) |

## 9. 확장 비전 (발표용)

### 콜라보 모델

```
지자체 콜라보
  → 삼척시 의뢰 공식 일러스트 카드
  → 지역 홍보 + 한정 카드 수집 유도

작가 콜라보  
  → 시즌 한정 일러스트 카드
  → 작가 크레딧 표시 + 작가 수익 모델 가능

축제 콜라보
  → 정동진 해돋이축제 한정 카드
  → 축제 기간에만 획득 가능 → 실제 축제 방문 유도
```

### 3자 윈윈 구조

```
수집가 (사용자)  → 한정 카드 수집 욕구 충족
지자체/축제     → 실제 관광객 방문 유도
작가/크리에이터 → 작품 노출 + 수익 기회
```

### 기술적 확장 용이성

카드 이미지가 URL 기반이므로, TourAPI 사진이든 콜라보 일러스트든 **URL만 교체하면 동작**한다.
별도 앱 업데이트나 코드 변경 없이 DB의 이미지 URL만 관리하면 된다.
