# 아이두(AI-DO) 페이지별 기능 정의

> 총 6페이지 | 모바일 퍼스트

---

## 전체 플로우

```
로그인 → 뽑기 → 체크리스트 → 스탬프 획득/공유 → 컬렉션(아카이브)
```

---

## 1. 로그인 (`login.html`)

**입력**:
- 닉네임 (2~10자)
- 출발역 선택 (드롭다운)

**동작**:
- 입력 완료 → JWT 발급 → 뽑기 페이지 이동
- 출발역은 가챠 풀에서 제외됨

---

## 2. 메인/뽑기 (`index.html`)

**표시 정보**:
- 남은 뽑기 횟수 (5회/일)
- 뽑기 결과: 역 이름, 노선, 등급(노말/레어/SSR)
- 동해산타열차 역이면 "환승" 표시

**버튼**:
- 뽑기 (횟수 차감)
- 다시 뽑기 (횟수 차감, 새 역)
- 미션 받기 → 체크리스트로 이동

**규칙**:
- 출발역 안 나옴
- 스탬프 찍힌 역 안 나옴
- 0회면 뽑기 비활성화
- 전체 수집 완료 시 뽑기 불가

---

## 3. 체크리스트 (`checklist.html`)

**표시 정보**:
- 역 이름 + 노선 + 등급 + 환승 여부
- 미션 목록 (3~5개, AI 생성)
- 각 미션: 제목 + 설명 + 체크박스
- 진행률 (2/3 완료)

**동작**:
- 첫 진입 시 SSE로 미션이 하나씩 나타남 (로딩 표시)
- 미션 탭 → 완료 토글 (체크 ↔ 언체크)
- 전부 완료 → "스탬프 받기" 버튼 활성화
- 스탬프 받기 → 공유 페이지로 이동

**규칙**:
- 미완료 상태로 나가도 체크리스트 유지됨
- 미완료 상태에서 새 뽑기 가능

---

## 4. 스탬프 공유 (`share.html`)

**표시 정보**:
- 스탬프 이미지
- 역 이름 + 등급 + 노선
- 획득일
- 완료한 미션 요약
- 닉네임

**버튼**:
- 사진 저장 (카드 영역을 이미지로 캡처)
- 컬렉션으로 이동

---

## 5. 컬렉션/아카이브 (`collection.html`)

**표시 정보**:
- 총 수집률 (12/62역)
- 노선별 구분
- 기찻길 느낌 배치
  - 획득한 역: 활성 표시, 탭하면 상세
  - 미획득 역: 잠금 표시 (역 이름은 보임)

**동작**:
- 획득 스탬프 탭 → 상세 보기 (역 이름, 등급, 획득일, 미션 내역)

---

## 공통

- 상단 네비: 홈(뽑기) / 컬렉션
- 컬렉션은 어느 페이지에서든 접근 가능
